# SkateGoal 交互逻辑设计规范 (v1.0)

本文档详细描述 SkateGoal 小程序各页面的完整交互逻辑、状态流转及反馈机制。适用于开发阶段直接对照实现。

## 全局交互规范 (Global Interactions)
*   **触感反馈 (Haptics)**：
    *   **轻微震动 (Light)**：常规点击，如展开折叠面板。
    *   **中等震动 (Medium)**：状态切换，如从“未解锁”变为“体验卡”。
    *   **重度震动 (Heavy/Success)**：成就达成，如点亮“一脚一个”。
*   **加载状态 (Loading)**：
    *   所有涉及网络请求的操作（如切换列表、加载更多），需展示骨架屏 (Skeleton) 或顶部加载条，避免全屏 Loading 阻断操作。
*   **视觉风格 (Theme)**：
    *   **暗黑模式 (Dark Mode)**：背景 `#121212`，卡片 `#1E1E1E`，文字 `#FFFFFF` / `#B0B0B0`。
    *   **主色调**：
        *   `体验卡`：`#4A90E2` (蓝)
        *   `死磕中`：`#FF5733` (燃橙)
        *   `一脚一个`：`#00C853` (荧光绿/金)

## 页面一：动作库 (Trick Library) - Tab 1

### 1. 页面结构
*   **顶部栏 (Sticky Header)**：
    *   **搜索框**：居中，占宽 80%。Placehoder: "🔍 搜个招..."。点击跳转搜索页。
    *   **分类筛选 (Filter Chips)**：横向滚动条。
        *   默认选中：`全部`
        *   选项：`平地 (Flat)`、`道具 (Obstacles)`、`弧面 (Transition)`、`必修 (Basics)`
*   **动作列表 (Trick List)**：
    *   **分组标题**：如 “--- 基础翻板 ---”，吸顶展示。
    *   **列表项 (Card)**：每个“根招式”为一个卡片。
        *   **左侧**：招式图标 (Icon) + 招式名称 (Kickflip)。
        *   **右侧**：脚位进度微缩图 (Mini Status Bar)。
            *   展示 `N` `F` `S` `No` 四个小字/圆点。
            *   **已点亮**：显示对应颜色高亮点（蓝/橙/绿）。
            *   **未解锁**：显示灰色低调点。
        *   **交互**：点击卡片任意区域 -> **从底部弹起“脚位矩阵面板”**。

### 2. 底部弹窗：脚位矩阵面板 (Stance Matrix Action Sheet)
*   **触发**：点击列表项。
*   **结构**：
    *   **标题栏**：招式名称 (Kickflip) + 关闭按钮 (X)。
    *   **矩阵区 (Grid)**：2x2 布局或 4行列表。
        *   **Normal (正脚)**
        *   **Fakie (倒滑)**
        *   **Switch (反脚)**
        *   **Nollie (反脚倒滑)**
    *   **每个脚位的交互单元**：
        *   **显示**：当前状态图标 + 文字。
        *   **操作**：点击某个脚位 -> **弹窗内层级切换** 或 **展开选项**。
            *   *(推荐交互)*：点击脚位区域 -> 立即展示 3 个状态选项（体验卡/死磕中/一脚一个）供选择。
*   **状态切换逻辑**：
    1.  用户点击 `Normal` 区域。
    2.  展开选项：`[🎫 体验卡] [🔥 死磕中] [🦶 一脚一个]`。
    3.  用户选择 `🦶 一脚一个`。
    4.  **反馈**：
        *   **视觉**：该区域瞬间变绿/金，播放撒花/盖章动画。
        *   **触感**：重度震动 (Success)。
        *   **数据**：更新本地及云端数据。
        *   **列表联动**：关闭弹窗后，列表页该卡片的 `N` 点变亮。

### 3. 搜索交互
*   **触发**：点击顶部搜索框。
*   **逻辑**：
    *   输入字符时（如 "ki"），实时过滤列表，仅显示匹配的“根招式”。
    *   支持中文/英文模糊匹配（如 "kf" -> Kickflip, "尖翻" -> Kickflip）。

## 页面二：我的 (Profile) - Tab 2

### 1. 页面结构
*   **个人信息卡片 (Header Card)**：
    *   **头像/昵称**：微信授权获取。
    *   **ID/站位/板龄**：小字展示。
    *   **统计概览**：
        *   `修练进度`：已点亮动作总数 / 总动作数（如 12/108）。
        *   `头衔`：当前佩戴的最高级徽章（如 [翻板大王]）。
*   **成就时光轴 (Timeline)**：
    *   **布局**：垂直时间轴。最新记录在最上。
    *   **节点 (Item)**：
        *   **日期**：`2024.03.15`
        *   **内容**：
            *   `[图标] 动作名 (脚位)`
            *   *例*：`🦶 Kickflip (Switch)`
        *   **视觉区分**：
            *   **一脚一个**：高亮，字号大，图标醒目。
            *   **体验卡**：普通亮度。
            *   **死磕中**：**不展示**在时光轴中（因为还没成）。
*   **底部版权/设置**：
    *   版本号 (v1.0.0)。
    *   “关于 SkateGoal”。

### 2. 交互逻辑
*   **下拉刷新 (Pull to Refresh)**：
    *   刷新时光轴数据，同步云端最新记录。
*   **点击时光轴节点**：
    *   *(可选)* 跳转到该动作的“脚位矩阵面板”，方便用户回顾或修改状态（比如不想练了，退回“死磕中”）。
*   **头衔点击**：
    *   弹出“头衔墙”弹窗，展示所有已获得和待解锁的头衔列表。

## 特殊状态与空态 (Edge Cases)

### 1. 首次进入 (Onboarding)
*   **引导**：
    *   全屏蒙层提示：“选择你的站位 (Regular / Goofy)”。
    *   *(重要)* 站位选择后，系统自动配置默认脚位逻辑（不影响数据结构，只影响显示顺序）。
*   **权限**：
    *   在点击“一脚一个”或进入“我的”页面时，触发微信登录授权（UserInfo）。若拒绝，则仅本地存储数据（Local Storage），清除缓存后丢失。

### 2. 断网状态 (Offline)
*   **读数据**：优先读取本地缓存（Storage），正常展示列表和状态。
*   **写数据**：
    *   允许用户操作修改状态。
    *   本地立即更新 UI（乐观更新）。
    *   后台将请求加入队列，网络恢复后静默同步。
